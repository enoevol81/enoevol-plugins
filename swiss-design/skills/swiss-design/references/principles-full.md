# Swiss Design (International Typographic Style) — Full Reference

Swiss design is not a visual style. It is a system for organizing information. The goal is clarity, hierarchy, objectivity, and reduction of noise. When making design decisions, prioritize communication over decoration.

This is the long-form rationale behind each tenet. For the operational version and the approval gate, see the parent `SKILL.md`.

-----

## 1. Grid Systems

Everything begins with the grid. A grid creates order, consistency, rhythm, and predictability. It allows complex information to feel effortless.

**Rules**

- Establish a grid before designing.
- Align all elements to columns, rows, and margins.
- Break the grid only with intention.
- Use whitespace as part of the grid structure.

**Ask**

- What invisible structure is governing this layout?
- Can another designer understand the system immediately?

![grid / layout levels](../assets/references/01-grid-layout-levels.webp)

## 2. Alignment

Nothing should appear randomly placed. Elements should share visual relationships through consistent alignment.

**Rules**

- Align edges whenever possible.
- Create strong vertical and horizontal axes.
- Avoid floating objects.
- Reduce arbitrary positioning.

**Ask**

- What line is this element aligned to?
- Does every object belong somewhere?

![alignment / vision](../assets/references/02-alignment-vision.jpeg)

## 3. Hierarchy

The viewer should know what to read first, second, and third. Hierarchy guides attention.

**Tools:** Scale · Weight · Position · Contrast · Spacing

**Rules**

- One dominant message.
- One supporting message.
- Everything else is secondary.

**Ask**

- Can someone understand this in three seconds?

![hierarchy / helvetica](../assets/references/03-hierarchy-helvetica.jpeg)

## 4. Typography First

Typography is the primary design tool. Swiss design often treats type as architecture rather than decoration.

**Preferred characteristics:** sans-serif typefaces · clear letterforms · consistent spacing · strong rhythm

**Rules**

- Let typography carry the communication.
- Avoid decorative fonts.
- Use fewer typefaces.
- Rely on scale and weight before adding effects.

**Ask**

- Would this still work if all imagery disappeared?

![typography / beethoven](../assets/references/04-typography-beethoven.jpeg)

## 5. Asymmetrical Balance

Balance does not require symmetry. Visual weight can be distributed unevenly while maintaining stability.

**Rules**

- Use asymmetry to create energy.
- Counter large elements with smaller elements and whitespace.
- Avoid perfectly centered compositions by default.

**Ask**

- Does the composition feel intentional rather than mirrored?

![asymmetry / die zeitung](../assets/references/05-asymmetry-die-zeitung.jpeg)

## 6. Contrast

Contrast creates hierarchy and focus. Without contrast, everything competes equally.

**Contrast can be created through:** size · weight · scale · color · position · density

**Rules**

- Make differences obvious.
- Avoid subtle distinctions that require explanation.
- Increase separation between important and unimportant content.

**Ask**

- What is the most important thing on the page?

![contrast / grid templates](../assets/references/06-contrast-grid-templates.jpeg)

## 7. White Space

White space is an active design element. It is not empty space. It creates breathing room, improves comprehension, and increases perceived quality.

**Rules**

- Remove unnecessary elements before adding new ones.
- Give important content room to exist.
- Use spacing to create relationships.

**Ask**

- What can be removed?
- What can be separated further?

![white space / velvet](../assets/references/07-whitespace-velvet.jpeg)

## 8. Consistency

Systems create trust. A viewer should learn the rules once and apply them everywhere.

**Rules**

- Consistent spacing.
- Consistent typography.
- Consistent grid usage.
- Consistent interaction patterns.

**Ask**

- If this element changes, should every similar element change too?

![consistency / wireframes](../assets/references/08-consistency-wireframes.webp)

## 9. Restraint

The most overlooked Swiss principle. Just because something can be added does not mean it should be. Swiss design values reduction over accumulation.

**Rules**

- Remove decorative noise.
- Eliminate redundant information.
- Prefer one strong idea over many competing ideas.
- Design with purpose.

**Ask**

- Does this element improve communication?
- If removed, does the design become stronger?

![restraint / swiss white](../assets/references/09-restraint-swiss-white.jpeg)

## Evaluation Checklist

Before approving any design:

- Is there a clear grid?
- Is everything aligned intentionally?
- Is the hierarchy obvious?
- Does typography carry most of the communication?
- Is the composition asymmetrically balanced?
- Is contrast being used effectively?
- Is whitespace doing meaningful work?
- Are design decisions consistent?
- Has every non-essential element been removed?

If any answer is "no," continue refining. The objective is not visual excitement. The objective is clarity, order, and effortless communication.
