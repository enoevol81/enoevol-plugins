# Swiss Design

Swiss design is not a visual style — it is a **system for organizing information**. The objective is clarity, hierarchy, objectivity, and the reduction of noise. Every decision answers one question: *does this improve communication?* If not, it goes.

When designing or critiquing, **prioritize communication over decoration**. Treat the nine tenets below as the operating system and the checklist at the end as a hard gate before declaring anything finished.

## Workflow (apply in order)

1. **Build the grid first.** Never place an element before the invisible structure exists. Define columns, rows, margins, gutters, and a baseline grid. The grid is the design decision; everything after is placement.
1. **Set type as architecture.** Place the dominant message, then the supporting message, then everything secondary. Use scale, weight, and position — not effects.
1. **Align everything to the grid.** No floating objects. Every element sits on a line and belongs to a column.
1. **Establish hierarchy through contrast.** Make the most important thing unmistakably the most important. Big differences, not subtle ones.
1. **Balance asymmetrically.** Counter mass with whitespace. Avoid reflexive centering.
1. **Cut.** Remove every non-essential element. Then run the checklist. If anything fails, keep refining.

## The Nine Tenets

### 1. Grid Systems — *everything begins with the grid*

Order, consistency, rhythm, predictability. A grid lets complex information feel effortless.

- Establish the grid before designing. Align all elements to columns, rows, margins.
- Break the grid only with intention. Whitespace is part of the grid, not leftover space.
- **Ask:** What invisible structure governs this layout? Could another designer read the system instantly?

### 2. Alignment — *nothing appears randomly placed*

Elements relate through consistent alignment. Strong vertical and horizontal axes.

- Align edges wherever possible. Kill arbitrary positioning. No floating objects.
- **Ask:** What line is this element aligned to? Does every object belong somewhere?

### 3. Hierarchy — *the viewer knows what to read first, second, third*

Hierarchy guides attention. Tools: scale, weight, position, contrast, spacing.

- One dominant message. One supporting message. Everything else is secondary.
- **Ask:** Can someone grasp this in three seconds?

### 4. Typography First — *type is the primary tool, treated as architecture*

Let typography carry the communication. Sans-serif, clear letterforms, consistent spacing, strong rhythm.

- Fewer typefaces. No decorative fonts. Reach for scale and weight before any effect.
- **Ask:** Would this still work if every image disappeared?

### 5. Asymmetrical Balance — *balance without symmetry*

Distribute visual weight unevenly while keeping stability. Asymmetry creates energy.

- Counter large elements with small ones and whitespace. Don't default to centered.
- **Ask:** Does the composition feel intentional rather than mirrored?

### 6. Contrast — *contrast creates hierarchy and focus*

Without contrast, everything competes equally. Build it through size, weight, scale, color, position, density.

- Make differences obvious. Avoid subtle distinctions that need explanation. Widen the gap between important and unimportant.
- **Ask:** What is the single most important thing on the page?

### 7. White Space — *an active element, not empty space*

Whitespace creates breathing room, improves comprehension, raises perceived quality.

- Remove before you add. Give important content room. Use spacing to create relationships.
- **Ask:** What can be removed? What can be separated further?

### 8. Consistency — *systems create trust*

A viewer learns the rules once and applies them everywhere. Consistent spacing, type, grid usage, interaction patterns.

- **Ask:** If this element changes, should every similar element change too?

### 9. Restraint — *the most overlooked principle*

Reduction over accumulation. Just because something *can* be added doesn't mean it *should* be.

- Remove decorative noise. Eliminate redundant information. One strong idea beats many competing ones.
- **Ask:** Does this element improve communication? If removed, does the design get stronger?

## Evaluation Checklist (the gate)

Before approving ANY design, every box must be checked. If any answer is "no," keep refining — do not present it as done.

- [ ] Is there a clear grid?
- [ ] Is everything aligned intentionally?
- [ ] Is the hierarchy obvious within three seconds?
- [ ] Does typography carry most of the communication?
- [ ] Is the composition asymmetrically balanced?
- [ ] Is contrast being used effectively?
- [ ] Is whitespace doing meaningful work?
- [ ] Are design decisions consistent?
- [ ] Has every non-essential element been removed?

The objective is not visual excitement. The objective is clarity, order, and effortless communication.

## When critiquing existing work

Walk the nine tenets in order and name the single highest-leverage fix first — usually grid, hierarchy, or restraint. Be specific ("the subhead competes with the headline; drop it two weights and align it to column 2"), not vague ("add more contrast"). End by running the checklist and stating which boxes currently fail.

## Applying within the Enoevol / Vanish system (optional)

This skill is style-agnostic, but when working inside the Enoevol/Vanish design language the tenets map cleanly onto the existing tokens — they don't override them:

- **Typography First / Restraint** → Inter for text, Geist Mono for labels/metadata; let weight and scale do the work.
- **Restraint / Consistency** → flat, `border-radius: 0`, no decorative shadows or gradients.
- **Contrast / Hierarchy** → coral `#ff5b45` is the single accent — use it to mark the one most important thing, never as decoration. Spend it sparingly or it stops meaning anything.
- **Grid / Alignment** → establish the column + baseline grid before placing components; align coral blocks and type to it.
