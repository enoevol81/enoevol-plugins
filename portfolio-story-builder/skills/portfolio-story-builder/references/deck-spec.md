# Deck Spec

Read this in Phase 5, alongside the `pptx` skill (which owns the *how* of
building the file — layouts, fonts, image placement, saving). This file owns the
*what*: the slide sequence and the design direction for a design portfolio deck.
Don't duplicate pptx mechanics here; defer to that skill for construction.

---

## Deck-level structure

```
1. Title slide          — name, role/positioning line, contact, one signature image
2. (optional) Index     — only if 5+ projects; a clean contents list
3..N. Project sequences — each project = a tight run of slides (below)
Final. Closing / contact
```

Order projects by the ranking the user approved in Phase 3 — strongest first,
strongest-second last, so the deck opens and closes on highlights.

## Per-project slide sequence

Aim for **4–6 slides per project**. More than that and it's a monograph, not a
portfolio. The arc from the case-study reference maps onto slides like this:

1. **Cover / hero** — full-bleed hero image. Overlaid: project title, one-line
   positioning, and a small metadata strip (year · role · client/brand). This
   single slide should make a viewer want the next one.
2. **Context** — the brief and the hard constraint, in as few words as possible.
   Optionally one reference or mood image. Resist paragraphs; a portfolio deck
   is shown while talking, not read in silence.
3. **Process** — the workflow proof. Sketches → 3D → samples laid out as a
   sequence so the viewer reads the evolution. This is the slide that documents
   what the user says they're bad at documenting; make it carry weight.
4. **Craft & decisions** — detail shots with short callouts on CMF, construction,
   tooling. One strong decision explained beats five listed.
5. **Resolution** — final hero shots, in-context / on-foot / on-body. Let the
   imagery be large; minimal type.
6. **Impact** (optional) — outcome, numbers, reception, or honestly-framed intent.

Collapse to 4 when a project is thinner; never pad to 6 with weak assets.

## Image handling

- Use the assets the user approved in Phase 4 — heroes as heroes, details as
  details. Don't resurrect culled or near-duplicate images.
- Favor large imagery and generous negative space over dense collages. A
  design portfolio is judged partly on the *layout's* taste, not just the work.
- If an approved image is low-resolution for the slot it needs to fill, flag it
  rather than upscaling into mush — note it in the gap list.

## Design direction

A portfolio deck's own visual language is part of the portfolio. Default to an
editorial, restrained system: big images, lots of white (or a single dark)
ground, one or two type sizes, tight alignment, no decorative clutter.

**Pull real brand tokens before inventing any.** If the user has brand
research, a color system, or type choices (e.g. from their own studio/magazine
work), ask for them or read them and use those — a portfolio that looks like the
designer's actual taste beats a generic template. Only fall back to a clean
neutral system if no brand direction is available, and say so when you do.

## Speaker notes

Put the longer narrative — the parts that would clutter a slide — into speaker
notes per slide, drawn from the case-study arc and the gap interview. The slides
stay visual; the story lives in the notes the user can speak from or adapt into
written copy later. This also makes the deck double as the workflow documentation
the user wanted in the first place.
