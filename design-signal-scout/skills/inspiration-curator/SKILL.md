# Inspiration Curator

Read [../../references/operating-model.md](../../references/operating-model.md) first — collections built here should carry the `inspiration` signal category and cite sources per the shared conventions.

## Objective
Turn scattered references into focused, useful inspiration collections tied to a project, theme, or question.

## Inputs
- reference images
- links
- videos
- project context
- desired mood
- target industry
- constraints

## Collection Types
Create:
- visual direction
- material direction
- form language
- color direction
- rendering direction
- motion direction
- UI direction
- manufacturing reference
- competitor reference
- cross-industry analogy
- experimental study

## Process
1. Define the question the collection should answer.
2. Remove redundant references.
3. Favor diversity over volume.
4. Group by meaningful attributes.
5. Explain why each reference is included.
6. Identify what should not be copied.
7. Translate references into design principles.
8. Suggest one concrete experiment.

## Curation Rules
A useful collection should:
- contain 6–15 strong references
- include multiple industries when helpful
- avoid near-duplicates
- include source attribution
- balance familiar and unexpected examples
- contain an explicit takeaway
- connect to a project or decision

## Output
```markdown
# Inspiration Set: [Theme]

## Question
What this set is meant to answer.

## Direction
One concise description of the visual or technical direction.

## Reference Groups

### Group 1: [Name]
- Reference
- Why it matters

### Group 2: [Name]
- Reference
- Why it matters

## Shared Principles
- ...

## Avoid
- ...

## Experiment
One test to run using this inspiration set.
```

## Rules
- Do not produce a mood board with no interpretation.
- Do not save references only because they look attractive.
- Avoid collections dominated by one creator or brand.
- Make the cross-industry connection explicit.
- Keep collections small enough to use.
